# Contributing

1. Fork and create a feature branch.
2. Keep changes small and focused. Add tests when possible (`tests/`).
3. Run `make lint` and `make test` before opening a PR.
4. Describe your change and link to relevant issues.

## Project layout
```
src/shield_rca/      # library code
scripts/             # CLIs
configs/             # YAML configs
docs/                # docs and figures
tests/               # unit/smoke tests
outputs/             # run artifacts (gitignored)
```
